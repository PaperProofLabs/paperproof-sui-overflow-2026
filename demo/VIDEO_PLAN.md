# Video Plan

## Format

- Screen recording, 4 to 5 minutes.
- English narration and English subtitles.
- No private keys, wallet recovery phrases, local paths with secrets, or API
  keys visible.
- First segment: pitch deck.
- Second segment: website, artifact pages, SDKs, and agent workflow evidence.

## Available Materials

- Demo production repository:
  [paperproof-demo-video](https://github.com/PaperProofLabs/paperproof-demo-video)
- Slides repository:
  [paperproof-slides](https://github.com/PaperProofLabs/paperproof-slides)
- Website:
  [https://paperproof.site/](https://paperproof.site/)
- SDK release pages:
  [npm](https://www.npmjs.com/package/@paperproof/sdk-ts),
  [PyPI](https://pypi.org/project/paperproof-sdk-py/),
  [crates.io](https://crates.io/crates/paperproof-sdk-rs)

## Suggested Timing

| Time | Segment |
| --- | --- |
| 0:00 - 1:30 | Slides: problem, architecture, Sui + Walrus, protocol vision |
| 1:30 - 2:30 | Website tour: Explore, artifact detail, version records |
| 2:30 - 3:30 | Operation: publish, add version, comment, or verify |
| 3:30 - 4:20 | Developer and agent surface: SDKs and PaperProof Skill |
| 4:20 - 5:00 | Closing: why durable protocol artifacts matter |

## Editing Notes

Use subtitles for the final narration. Keep wallet popups and transaction
screens brief, but leave enough time for package IDs, artifact versioning, and
Walrus/Sui references to be visible.
